# Project 77 — Roadmap: Day 15 → Day 77
### Cybersecurity, from zero to employable/interviewing, on a zero budget

**Start date:** July 20, 2026 (Day 1)
**Deadline:** October 5, 2026
**Days 1-14 (Jul 20 - Aug 2):** Math phase (for CP/DSA) — not covered in this doc
**This roadmap covers Day 15 (Aug 3) → Day 77 (Oct 4)** — the cybersecurity grind itself

**Constraints going in:** zero budget for certs/exams, remote-only, zero prior hands-on security work, zero projects, knows Python (not interested in web dev, dropped MERN). Cybersecurity is a genuine calling, not a random pick. Baseline effort: ~12 hrs/day, sometimes 16 on specific days.

**Honest framing (read this before you start):** 63 days from zero to a paying/offer-stage cyber job is a stretch goal, not a guarantee — most entry cyber roles still favor people with certs or more hands-on hours than this window allows. The realistic bar is "actively interviewing with real proof-of-work by Oct 5," not "guaranteed employed by Oct 5." Treat every date below as a target, not a promise.

## PHASE 2: Days 25-49 (Aug 13 - Sep 6) — Depth + Python-Cyber Overlap

**Theme:** go from "completed beginner paths" to "has a real skillset + a visible portfolio." This is the longest phase — treat it as the core of the whole roadmap.

**Weekly structure (repeats with light days on 27, 34, 41, 48):**
- **THM track:** finish "Cyber Security 101," then move into THM's "SOC Level 1" learning path (directly relevant to SOC analyst roles) or "Jr Penetration Tester" intro rooms — pick based on which sub-field is pulling you more by Day 30.
- **Security+ track:** finish professormesser's full video series by ~Day 35. After that, do practice questions (free sets exist online) instead of paid exam vouchers — you're proving the knowledge, not the paper cert, given zero budget.
- **Python track:** build out 2-3 more security-relevant scripts — e.g. a brute-force login detector, a basic packet sniffer (scapy), a simple vulnerability-checker against a test VM. Every script gets pushed to GitHub with a README explaining what it does and why.
- **Home lab track:** expand beyond the basic setup — add a second vulnerable VM, try a basic pfSense firewall setup, start doing simple THM "practice" rooms that simulate real attack/defend scenarios.
- **Documentation:** daily notes into the GitHub repo, non-negotiable. This log is what makes 63 days of work visible to an employer instead of invisible.

**Key milestones inside Phase 2:**
- **Day 35 (~Aug 24):** Security+ material fully covered (video-watched, not exam-certified).
- **Day 42 (~Aug 30):** THM Cyber Security 101 complete; SOC Level 1 or Jr Pentester path underway.
- **Day 49 (Sep 6):** 4-5 Python security scripts live on GitHub, home lab meaningfully more complex than Phase 1, THM profile shows consistent activity.

**Light/rest days:** 27, 34, 41, 48 — same rule as Day 20. Review + light catch-up only, protect these.

## PHASE 3: Days 50-63 (Sep 7 - Sep 20) — Proof-of-Work Finalization + Start Applying

**Theme:** stop only building, start showing it to the world. This is the phase most people delay too long — don't.

- **Days 50-53:** Finalize 2-3 flagship Python security projects (polish, document, make sure they actually run cleanly for a stranger cloning the repo).
- **Days 50-53 (parallel):** Build LinkedIn properly — headline mentioning cybersecurity + Python, pin GitHub projects, write a short "what I've been building" post.
- **Day 54 onward: start applying, don't wait to feel ready.** Targets: SOC Tier 1, junior security analyst, security-adjacent IT/helpdesk roles, MSSPs (managed security service providers — they hire green candidates more than enterprises do).
- **Days 54-63 (parallel):** List Python scripting/automation gigs on Upwork/Fiverr (log parsers, small automation scripts) — this is your actual near-term income lever, separate from the cyber job hunt.
- **Light day:** 55, 62.

**Phase 3 checkpoint (end of Day 63):** Applications actively going out, 2-3 strong GitHub projects live, freelance gigs posted (ideally with first client interest), LinkedIn presence built.

---

## PHASE 4: Days 64-77 (Sep 21 - Oct 4) — Volume, Network, Close

**Theme:** this phase is about consistency and follow-through, not new skills.

- **Daily:** job applications — no gap days here, volume matters at entry level.
- **Daily/weekly:** engage on r/cybersecurity, r/AskNetsec — ask real questions, answer where you can, don't just lurk. Referrals beat cold applications at entry level.
- **Every application:** follow up after 5-7 days if no response.
- **Ongoing:** keep freelance Python gigs running for actual cash flow during this stretch.
- **Interview prep:** once any interviews land, prep specifically — SOC/security fundamentals, walk-throughs of your own GitHub projects (know them cold, be ready to explain every design decision).
- **Light days:** 69, 76.

**Day 77 (Oct 4) — final checkpoint before the Oct 5 deadline:** By here, the honest goal is: strong portfolio, consistent application volume, at least some interviews in progress, freelance income flowing. Whether a signed offer lands exactly on Oct 5 depends on employer hiring speed, which isn't fully in your control — but everything that *is* in your control will have been done.

---

## Non-negotiables across all 77 days
1. **Daily GitHub log** — even a rough entry. This is your proof-of-work and it's what separates you from every other applicant who "studied" but has nothing to show.
2. **Protect the light/rest days** — 20, 27, 34, 41, 48, 55, 62, 69, 76. Skipping them doesn't speed you up, it risks a burnout crash mid-plan.
3. **Apply before you feel ready** — Day 54 onward, applications go out regardless of how "finished" you feel.
4. **Re-check reality every 2 weeks** — are you actually done with what a given phase says, or behind? Adjust the remaining days honestly instead of pretending the plan is on track when it isn't.
